Top Repo
ByteDance releases DeerFlow 2.0, an open-source system where each AI agent runs tasks in its own isolated environment
6,468 Likes

DeerFlow 2.0 is a multi-agent framework that lets you run complex workflows through coordinated agents instead of one overloaded model. It solves shared-context issues by isolating each agent with its own memory, tools, and execution environment.

You give one prompt, and the system plans, executes, and returns a complete result.

What it does 
You send a task, and a lead agent breaks it into steps. Sub-agents handle each step in parallel, then return structured outputs that get merged.

  • Lead agent decomposes tasks and assigns scoped work to sub-agents
  • Sub-agents run independently with isolated context and tool access
  • Docker sandbox provides filesystem, bash execution, and code runtime
  • Skills load on demand, defined as simple Markdown workflows

How to use it 
You clone the repo, configure API keys, and run via Docker or locally.

  • Supports GPT, Claude, Gemini, DeepSeek through OpenAI-compatible APIs
  • Requires models with tool-calling and long context for multi-step tasks
  • Add custom tools via MCP servers or Python functions
  • Use built-in skills for research, reports, slides, and web generation

Top Paper
Meta publishes V-JEPA 2.1 that predicts every part of video for better understanding of scenes
3,621 Likes

Yann LeCun and team introduce V-JEPA 2.1, a self-supervised model for images and videos. It predicts every patch across frames and multiple layers instead of only masked regions.

This change improves how the model understands what is where over time, with a +20 point increase in robotic grasp success rate.

What changed

The model shifts from partial prediction to full-scene learning.

  • Predicts every patch, not just masked ones, improving spatial coverage across frames
  • Applies supervision at multiple layers, not only final outputs, improving feature depth
  • Uses shared tokenization for images and videos, enabling one unified training pipeline

How it works

The model learns representations without labels using prediction in latent space.

  • Convert frames into patch tokens and mask parts of the input sequence
  • Predict latent features for all tokens instead of reconstructing pixels directly
  • Apply loss across all tokens and intermediate layers for stronger supervision

OpenAI adds container pooling to Responses API, reusing environments and reducing setup latency for agent workflows
2,148 Likes

OpenAI updates the Responses API with container pooling, which reuses execution environments across requests. Instead of starting a new container each time, the system routes work to an existing one.

This cuts setup time by about 10x and keeps workflows moving without repeated initialization.

What it does
Reuses a running container so tools execute without rebuilding the environment each time.

  • A container is a lightweight runtime with preinstalled languages and tools
  • Requests reuse this runtime instead of creating a new one for every step
  • The environment stays active briefly, then expires after inactivity

Key features 
Focuses on reducing repeated setup and maintaining continuity across steps.

  • Reuses warm containers, removing repeated environment startup across agent tool calls
  • Supports shell, code interpreter, and skills in the same running environment
  • Keeps files so later steps can access earlier outputs
  • Runs on a Debian-based system with Python, Node.js, Java, and more
