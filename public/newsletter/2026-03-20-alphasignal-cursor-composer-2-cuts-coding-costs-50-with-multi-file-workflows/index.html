Cursor introduces Composer 2 with updated pricing tiers and a fast mode for lower latency execution
11,458 Likes

Cursor releases Composer 2, an updated coding model inside its IDE designed for multi-step development tasks. It combines continued pretraining (training further on large datasets) with reinforcement learning (learning from trial-and-error on coding tasks).

The model focuses on solving longer workflows that involve many actions across files, terminals, and tools.

Composer 2 targets developers who need an AI to handle extended coding tasks instead of single prompts. It runs directly inside Cursor and works across refactoring, debugging, and multi-file edits. The update improves benchmark performance while reducing token pricing.

Key features and results

  • Scores 61.3 on CursorBench, up from 44.2 in Composer 1.5 baseline

  • Reaches 61.7 on Terminal-Bench 2.0 for terminal-based coding task execution

  • Achieves 73.7 on SWE-bench Multilingual for multi-language bug fixing tasks

  • Uses continued pretraining to improve base model before reinforcement learning stage

  • Trains on long-horizon tasks requiring hundreds of sequential coding actions

  • Prices at $0.50/M input and $2.50/M output tokens in standard mode

  • Offers fast mode at $1.50/M input and $7.50/M output tokens

Google launches AI Studio update that generates full-stack apps with frontend, backend, and integrations from prompts
10,621 Likes

Google introduces a full-stack vibe coding system in AI Studio using the Antigravity agent and Firebase. You write prompts, and it generates working apps with frontend, backend, and integrations in one place.

The key change is a persistent agent that tracks your project and continues work across sessions.

What it does

It converts natural language into complete, runnable applications.

  • Generates UI (what users see) and backend (server logic) in one workflow

  • Supports real-time features like multiplayer state and live data updates

  • Connects apps to external services like maps, databases, or payments

  • Stores API keys securely using a built-in secrets manager

Why it matters

It removes manual setup across multiple tools and services.

  • Eliminates separate setup for hosting, database, and authentication systems

  • Reduces need to switch between frontend, backend, and infrastructure tools

  • Handles dependency installation and framework setup automatically

  • Maintains project context to reduce repeated instructions

Anthropic runs Claude-powered interviews with 80K users to map real-world AI usage patterns and concerns
6,728 Likes

Anthropic released a study where Claude runs structured interviews with 80,508 users across 159 countries and 70 languages to understand real AI usage and concerns. Instead of surveys, Claude asks questions and follows up based on answers, which captures detailed, open-ended responses.

The key idea: use an LLM as both interviewer and analyst to collect and structure qualitative data at scale.

Key findings 

The study shows how users apply AI across work, learning, and daily tasks.

  • 81% report AI already helped them move toward their goals

  • Productivity leads at 32%, mainly for coding and technical work acceleration

  • Cognitive support (17.2%) includes reasoning help and problem-solving assistance

  • Learning (9.9%) reflects self-paced education with AI as a tutor

Why it matters

This system replaces static surveys with adaptive AI-led interviews.

  • Claude Interviewer asks core questions, then generates follow-ups based on each response

  • Classifiers group answers into goals, concerns, and sentiment categories automatically

  • Human reviewers validate outputs and remove identifying information from responses

  • The setup captures “why” users behave a certain way, not just metrics
