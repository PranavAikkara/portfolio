Anthropic releases Claude Opus 4.7 improving long-running tasks, instruction following, and self-verification
106,283 Likes

Anthropic introduces Claude Opus 4.7 as an upgrade to Opus 4.6, focused on handling complex, long-running work with less supervision.

Earlier models required close oversight because they could skip steps or return unverified outputs. Opus 4.7 changes this by following instructions more strictly and checking its own work before responding.

The key shift is deeper reasoning with controlled effort. The model adds an xhigh effort level, which increases reasoning depth while balancing latency.

It also improves vision by supporting images up to 2,576 pixels (~3.75 MP), enabling accurate reading of dense screenshots and diagrams.

This extends to real workflows across coding, analysis, and document generation. The model also uses file-based memory to retain context across sessions and continue tasks without repeated input.

Key features

  • Handles long-running tasks with consistent step execution and internal verification
  • Follows instructions precisely, changing behavior compared to earlier prompt assumptions
  • Processes higher-resolution images for detailed multimodal understanding and extraction
  • Adds task budgets to control token usage across extended agent workflows
  • Improves performance on GDPval-AA and finance agent evaluations

How to use

  • Call claude-opus-4-7 in the API for advanced reasoning tasks
  • Use /ultrareview in Claude Code to analyze code changes systematically
  • Set effort to high or xhigh for complex multi-step problems

OpenAI expands Codex from a coding assistant into a system-level agent on macOS. You can now use it to control apps, run tasks, and manage workflows in one place.

What it does

Codex operates software the same way you do through the interface.

Alibaba unveils Qwen3.6-35B-A3B, a sparse MoE model activating only 3B parameters per token for efficiency
12,398 Likes

Qwen releases Qwen3.6-35B-A3B as an open-source model designed to reduce compute without reducing capability. Large models usually scale performance by increasing parameters, which raises cost and latency.

This release tests a different approach using a sparse mixture-of-experts design, where only a small subset of parameters runs per token.

The key idea is simple: instead of using all parameters every time, the model selects a few relevant “experts” for each step. This keeps 35B total parameters but only 3B active, lowering compute while maintaining performance.

That design shows up clearly in results across coding, agents, and multimodal tasks.

  • SWE-bench Verified reaches 73.4 using tool-based agent evaluation setup
  • Terminal-Bench 2.0 scores 51.5 under multi-run execution conditions
  • RefCOCO hits 92.0, measuring spatial reasoning in vision-language tasks
  • OmniDocBench reaches 89.9 for document understanding benchmarks

You can use the model through API, local weights, or agent frameworks.

  • Call qwen3.6-flash via Alibaba Cloud Model Studio API endpoint
  • Download weights from Hugging Face or ModelScope for self-hosting setups
  • Integrate with OpenClaw, Claude Code, or Qwen Code for agent workflows
  • Enable reasoning with enable_thinking in API requests for structured outputs
