Cursor updates Agents Window with persistent layouts, smoother UI behavior, and scoped extension events per workspace
3,539 Likes

Cursor updates its Agents Window to improve how you run and manage coding agents. The release introduces a tiled layout that lets you run multiple agents in parallel instead of switching tabs.

It also improves editing performance by reducing dropped frames during large file changes by about 87%.

What changed and why

Cursor focuses on reducing friction in multi-agent workflows and long editing sessions.

  • Run multiple agents side by side using a tiled layout for parallel tasks
  • Move from diff view directly to exact file lines for faster code inspection
  • Select branches before agent runs to avoid executing on the wrong codebase
  • Use filters in search to scope results to specific files or directories

How to use it

Cursor keeps the workflow close to how you already use an editor.

  • Split the Agents Window into panes and assign an agent to each tile
  • Press and hold Ctrl+M to dictate prompts using voice input
  • Launch agents after selecting a target branch from the empty state
  • Navigate diffs and jump into files to edit using standard editor shortcuts

System and interface updates

Cursor improves responsiveness and stability across long sessions.

  • Stream large edits more smoothly after reducing dropped frames significantly
  • Send follow-up messages instantly without delay in long conversations
  • Reduce UI flicker and stale updates in the file tree
  • Limit unnecessary background updates to lower CPU and network usage

Top News
Vercel provides a template for running chat-driven coding agents with persistent workflows and GitHub integration support
3,349 Likes

Vercel releases Open Agents as an open-source template to build coding agents that run in the cloud instead of your laptop. It targets teams building internal systems like those at Stripe, Ramp, Spotify, and Block.

The key idea separates the agent “brain” from the sandbox where code runs, so tasks continue independently and resume later.

How it works

The system splits responsibilities across interface, logic, and execution layers.

  • Web app handles login, sessions, chat UI, and streaming responses
  • Agent runs as a durable workflow, meaning tasks persist across multiple steps
  • Sandbox VM executes code with filesystem, shell, git, and dev servers
  • Agent uses tools to interact with sandbox instead of running inside it

Key features

The platform covers the full coding workflow from prompt to pull request.

  • Chat-driven agent edits files, runs shell commands, and searches codebases
  • Multi-step execution persists state, supports streaming updates, and cancellation
  • Sandboxes clone repos, create branches, and run isolated development tasks
  • Optional auto-commit, push, and PR creation after successful runs

How to use it

You fork the repo, configure environment variables, and deploy on Vercel.

  • Set POSTGRES_URL and JWE_SECRET to initialize server state
  • Add ENCRYPTION_KEY and Vercel OAuth credentials for user authentication
  • Start a chat to trigger a workflow run instead of local execution
  • Reconnect anytime to resume active runs without restarting processes

Top Paper
Google gives RNNs growing memory so they can handle longer inputs without large compute increases
2,431 Likes

Google Research proposes Memory Caching (MC), a method that extends RNNs with memory that grows over time.

Transformers solve long-context tasks using KV-cache but require O(L²) compute. Standard RNNs run in O(L) but compress all context into one state. MC stores selected hidden states so memory scales with sequence length while keeping per-token compute close to constant.

Core idea

MC replaces fixed memory with a checkpoint-based mechanism.

  • Stores intermediate hidden states as reusable memory checkpoints across the sequence
  • Enables access to earlier context without recomputing full token interactions
  • Uses learned selection to decide which past states remain available
  • Expands usable context length without increasing computation per step

What it changes

MC alters how recurrent models handle long-range dependencies.

  • Introduces gated aggregation to combine relevant cached states during prediction
  • Avoids full attention while still retrieving useful historical information
  • Keeps decoding cost stable even as sequence length increases
  • Supports longer context handling without redesigning the core recurrent update
