Anthropic adds classifier-based auto mode to reduce manual approvals while filtering risky actions during execution
35,285 Likes

Claude introduces auto mode in Claude Code, a new setting where the model decides whether to run file writes and shell commands. It removes the need to approve every action while avoiding fully skipping safeguards.

You can now run longer coding tasks without constant prompts, while a system checks each step before execution.

What it does

Auto mode handles permission decisions using a classifier that filters actions.

  • Reviews each command before execution to detect destructive or sensitive operations
  • Allows safe actions to run automatically without manual approval during sessions
  • Blocks risky actions and forces the model to try a different method
  • Prompts the user only after repeated blocked attempts

How to use it

You enable and switch modes directly inside Claude Code environments.

  • Run claude --enable-auto-mode to activate auto mode in CLI sessions
  • Press Shift+Tab to cycle through available permission modes
  • Available in Team plan as research preview, expanding to other plans
  • Use in isolated environments due to incomplete risk detection coverage

Top News
Figma introduces AI agents that design directly inside canvas using MCP tools and structured skills
9,727 Likes

Figma released a new use_figma tool in open beta that allows AI agents to create and edit designs directly inside Figma files. Earlier, agents generated code or static UI, but could not work inside real design systems.

Now, agents access components, variables, and layouts, so outputs stay consistent with how teams already design.

Key features

This adds direct design execution capabilities to AI agent workflows.

  • Agents operate on real Figma files, not exported or generated mockups
  • Works with MCP clients like Claude Code, Codex, Cursor, and Copilot CLI
  • generate_figma_design converts HTML into editable Figma layers
  • Supports multi-agent workflows across design and implementation tasks

How it works

MCP (Model Context Protocol) connects tools so agents can act on real systems.

  • use_figma lets agents create or modify layers, components, and layouts
  • Skills (markdown instructions) define steps, rules, and design conventions
  • Base /figma-use skill teaches agents how Figma structure works
  • Agents follow these rules to produce consistent, structured outputs

Top Paper
Google introduces TurboQuant, improves LLM efficiency with KV cache compression
11,486 Likes

LLMs slow down as context grows because the KV cache stores every token and consumes memory fast.

Google Research introduces TurboQuant to fix this bottleneck without changing model behavior. It cuts KV cache size by ≥6x and speeds up attention by up to 8x on H100 GPUs, while keeping accuracy unchanged.

The problem sits in how models store vectors, which are high-dimensional arrays that represent meaning. Standard compression reduces size but adds error or overhead.

TurboQuant splits the task: compress most information first, then fix the small errors.

The breakthrough combines two methods:

  • PolarQuant converts vectors into magnitude and direction for efficient storage
  • QJL stores residual errors as simple sign bits and corrects distortion
  • Runs at ~3–4 bits per value instead of 16-bit or 32-bit formats
  • Requires no retraining or fine-tuning on existing models

It changes how you run long-context models and vector search systems:

  • Reduces memory pressure in long conversations and agent workflows
  • Improves attention compute speed directly, not just storage efficiency
  • Maintains accuracy across LongBench, RULER, and ZeroSCROLLS benchmarks
  • Works with models like Gemma and Mistral without architecture changes
