Anthropic adds computer use to Claude Code, enabling full UI testing and debugging directly from the CLI
60,482 Likes

This update removes the gap between writing code and manually testing it by letting Claude interact with the UI itself. In one prompt, it can build, run, test, and fix an app while observing the screen.

What it does

This feature lets Claude control your computer like a user would.

  • Opens apps, clicks buttons, types inputs, and reads results from the screen
  • Runs UI-based workflows that do not expose APIs or command-line interfaces
  • Uses screenshots to understand outputs instead of relying only on logs
  • Handles GUI tasks that normally require switching out of the terminal

Key features

These capabilities focus on end-to-end development and testing workflows.

  • Builds apps, launches them, and verifies flows without external test frameworks
  • Runs onboarding or user flows in Electron and native macOS apps
  • Reproduces visual bugs like layout issues by resizing and interacting with UI
  • Fixes code and validates changes using screenshots in the same session

How to use it

You enable and run it inside the CLI with minimal setup.

  • Install Claude Code version 2.1.85 or later on macOS systems
  • Enable computer use using the /mcp command in an interactive session
  • Available in research preview on Pro and Max plans only

Alibaba ships Qwen3.5-Omni, surpassing Gemini 3.1 Pro on audio reasoning and speech benchmarks
5,427 Stars

Alibaba launches Qwen3.5-Omni, a model that handles text, images, audio, and video in one system. You send mixed inputs and receive text or speech outputs through the same interface.

The key capability is audio-visual coding, where you describe an idea with voice or camera and the model generates working apps or games.

Key Features 

Core capabilities focus on multimodal reasoning and real-time interaction.

  • Surpasses Gemini 3.1 Pro on audio reasoning and speech benchmarks
  • Supports 256K token context across combined multimodal inputs in one request
  • Processes 10+ hours audio and 400 seconds 720p video without splitting inputs
  • Trained on 100M+ hours of multimodal data across speech, video, and text

You can choose between three variants. Plus targets maximum capability for reasoning and long-context workloads. Flash targets lower latency for real-time systems. Light targets smaller deployments with reduced cost.

Microsoft enables Copilot to use Critique for reviewing outputs and Council for comparing multiple model responses
6,249 Likes

Microsoft adds Critique and Council to Microsoft 365 Copilot Researcher, enabling multi-model research workflows. The system combines models from OpenAI and Anthropic to generate and evaluate reports together.

The key change is a split between generation and review, where separate models handle drafting and validation.

Critique: dual-model generation and evaluation

Critique assigns distinct roles to two models in the research pipeline.

  • One model plans tasks, retrieves sources, and produces the initial research draft
  • Second model reviews output for source reliability, completeness, and evidence grounding
  • Uses rubric-based evaluation aligned with structured academic review processes
  • Enforces strict citation checks to ensure every key claim links to sources
  • Achieves +7.0 DRACO score improvement across 100 evaluated research tasks

Council: parallel model comparison

Council runs multiple models simultaneously on the same input prompt.

  • Generates independent reports from each model with separate reasoning and citations
  • Uses a judge model to compare outputs and summarize agreement and divergence
  • Highlights differences in framing, interpretation, and evidence selection across models
  • Surfaces unique insights contributed by each model in the final comparison layer
  • Exposes how models differ on identical research tasks without merging outputs
