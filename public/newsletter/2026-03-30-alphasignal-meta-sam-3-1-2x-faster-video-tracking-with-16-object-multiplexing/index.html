Top News
Meta increases video throughput and reduces compute overhead with multiplexed object tracking in SAM 3.1
2,474 Likes

Meta releases SAM 3.1 as an update to its Segment Anything Model for image and video segmentation. It focuses on tracking objects across video frames using prompts like text, points, or masks.

The key change is object multiplexing, which lets the model process multiple objects together instead of separately.

Video tracking usually runs one pass per object, which increases compute cost as objects increase. SAM 3.1 removes this by grouping objects into a single pass. This reduces repeated work and simplifies multi-object tracking pipelines.

Key features

  • Tracks up to 16 objects in one pass using shared computation across frames

  • Increases throughput from 16 to 32 FPS on a single H100 GPU

  • Removes redundant per-object passes that previously repeated feature extraction

  • Maintains segmentation and tracking accuracy with the new computation structure

Performance Changes

Improves video processing speed and reduces compute overhead in multi-object scenes.

  • Increases throughput from 16 to 32 frames per second on one H100 GPU
  • Reduces memory bottlenecks caused by repeated per-object inference pipelines
  • Maintains accuracy while changing the computation structure for object tracking
  • Enables real-time processing in videos containing a medium number of objects

Supermemory replaces vector databases with a structured memory graph for consistent context across AI conversations
20,427 Stars

Supermemory is an open-source memory layer for AI systems that replaces vector databases and embedding pipelines with a structured memory graph.

It solves a common issue: AI systems forget past interactions or return inconsistent context across sessions.

The system reports 81.6% on LongMemEval and ranks first on LoCoMo and ConvoMem, which measure long-term recall, reasoning over time, and personalization.

Key Features and Results
Supermemory stores and retrieves context as structured facts instead of text chunks.

  • Extracts facts from conversations and organizes them into entities, relationships, and timestamps
  • Updates or replaces outdated or conflicting information using recency and contextual rules
  • Combines memory and document retrieval in one query instead of separate RAG pipelines
  • Returns user profile context in ~50ms without scanning full conversation history
  • Uses multi-agent retrieval (ASMR) to read, verify, and reconstruct context across sessions

How to Use It
You add memory to agents through a single API or MCP server integration. Install via npm or pip and store context with simple API calls

Princeton finds reasoning models consume 79× more energy per query due to longer compute chains
1,727 Likes

Princeton University researchers analyze how modern AI systems scale in energy, cost, and reasoning. They estimate a next-gen model training run uses 11 billion kWh, while a single reasoning query uses 33 Wh vs 0.42 Wh, a 79× jump.

The paper argues current large models rely on pattern matching outside math and code, which limits real reasoning.

Key findings

The paper quantifies where current systems break and what works instead.

  • Energy shifts from training to inference due to repeated query execution at scale
  • Reasoning models increase compute through longer token chains and multi-step generation
  • Performance stays strong in structured domains but drops in unstructured real-world tasks
  • A 14B domain-trained model beats a larger general model in clinical reasoning benchmarks
  • Local specialist models achieve ~10,000× lower energy per query than cloud reasoning systems

How to use this approach

The paper explains how you can build systems like this today.

  • Train small models (1B–15B parameters) on focused datasets from one domain
  • Build knowledge graphs, which store facts and relationships in a structured format
  • Add a router that sends each query to the right specialist model
  • Run models locally with optimized runtimes instead of relying on cloud GPUs
